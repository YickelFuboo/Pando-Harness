# AI4SE 资讯洞察总控

当用户提出“围绕某主题做资料分析并形成洞察报告”时，优先使用本技能统筹执行。

## 目标

- 把复杂信息任务拆成可执行阶段并稳定交付
- 在“信息摘要”基础上，输出“高价值洞察+行动建议”
- 保证报告与用户业务关注点强相关

## 推荐流程

1. 任务定义  
   明确主题、受众、时效、输出格式与深度。

2. 信息获取  
   优先处理用户指定材料；不足时自主检索补齐。

3. 摘要分析  
   调用 `key-content-summary-analysis` 提炼核心事实与多源差异。

4. 报告生成  
   调用 `jiaxian-report-generation` 按模板产出可交付报告。

5. 结果复核  
   校验完整性、证据一致性、启发价值和可执行性。

## 关键输出要求

- 先结论后展开
- 事实、观点、建议分层清晰
- 结合 AI4SE 与通信行业应用语境
- 明确“机会、风险、下一步行动”

## 模板资源

- 参考模板：`reference/洞察报告模板.md`
- 若模板与用户指定格式冲突，以用户要求优先
