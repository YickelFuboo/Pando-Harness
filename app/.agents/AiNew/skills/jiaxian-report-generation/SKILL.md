# jiaxian 报告生成

当任务目标是“产出可交付的洞察分析报告”时，使用本技能。

## 适用场景

- 用户明确要求输出 jiaxian 报告/洞察报告
- 已有摘要结果，需要整理成完整报告
- 需要按固定模板产出并保持风格稳定

## 强约束

- 优先使用模板：`skills/AI4SE-News-Report/reference/洞察报告模板.md`
- 若用户指定格式，以用户格式优先；否则严格按模板章节输出
- 结论必须与材料证据一致，不夸大、不脑补
- “思考与讨论”需贴合用户工作方向：  
  AI 在软件研发工程的应用、Agent 架构实践、通信行业 AI 智能化演进

## 执行步骤

1. 读取输入  
   接收用户给定材料，或接收 `key-content-summary-analysis` 的输出结果。

2. 结构映射  
   将素材映射到模板章节：标题、关键词、主要内容、思考与讨论。

3. 深度加工  
   从“信息罗列”提升到“洞察表达”，补充趋势、影响、风险、机会。

4. 行动化输出  
   在“思考与讨论”中给出可执行建议：短期试点、中期能力建设、长期演进方向。

5. 完整性校验  
   检查是否满足模板约束、是否有证据支撑、是否对用户工作有启发。

## 输出模板（默认）

# 标题
一句话概括报告主旨，控制在 25 字以内，突出关键点与关键字。

# 关键字
1-5 个关键词，使用顿号分隔。

# 主要内容
可按章节或条目输出，保证结构清晰、重点明确。

# 思考与讨论
围绕用户工作语境给出启发与建议，建议包含：
- 对当前工作的直接启发
- 可验证的实践方向
- 潜在风险与规避策略

## 质量门槛

- 报告结构完整，格式稳定
- 观点与证据对应，避免“结论先验”
- 不只总结现象，还要解释原因与影响
- 建议具备可执行性（对象、动作、预期结果尽量明确）
